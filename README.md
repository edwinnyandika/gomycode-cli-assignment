## What is DevStart?

<img align="right" width="380" src="https://images.stockcake.com/public/6/2/5/625e3a49-30e3-4b8f-8fac-2e13dca37fb8_large/glowing-code-terminal-image_stockcake.jpg" alt="DevStart terminal interface" style="border-radius:8px;margin-left:20px"/>

DevStart started as a **GoMyCode bootcamp CLI assignment** — document nine terminal commands, push to GitHub, done. It became something bigger.

It's now a full interactive learning platform covering the three pillars every developer needs before anything else: **the command line**, **Git**, and **HTML**. Everything — 8 pages of content, games, simulators, flashcards, quizzes, a live code editor, and a dictionary — ships as a **single `index.html` file** with no dependencies, no build step, and no server required.

**Why single-file?**
- Email it to someone. They open it. It works.
- No `npm install`. No Webpack. No config.
- Deploy anywhere that serves static files — GitHub Pages, Netlify, a USB stick.

```
Clone  →  Open  →  Learn.
That's the whole process.
```

Built with the dark terminal aesthetic of the tools developers actually live in. Black background, neon green accents, JetBrains Mono throughout.

## Pages in Detail

### ⌨️ CLI Tasks

<img src="https://images.stockcake.com/public/6/2/5/625e3a49-30e3-4b8f-8fac-2e13dca37fb8_large/glowing-code-terminal-image_stockcake.jpg" width="100%" alt="CLI terminal interface" />

Nine filesystem operations documented with the exact commands run during the bootcamp assignment. Each task shows the command, purpose, and expected terminal output — tested on both **Git Bash (Windows)** and **Termux (Android)**.

A live **interactive terminal simulator** runs in the browser. Type real commands against a virtual in-memory filesystem — no actual terminal needed.

```bash
# Commands available in the live simulator
mkdir    touch    rm     mv      cp
echo     cat      cd     ls      pwd
```

### 📖 Lexicon

Every term ships with **two definitions** — one plain-English, one technically precise — plus a syntax-highlighted code example and an SVG visual.

```
Terms covered:
mkdir · touch · rm · git init · git commit · git branch
HTML Tag · Attribute · DOM · CSS Class · Variable
Function · Repository · Flexbox · API
```

The search bar connects live to **`api.dictionaryapi.dev`** — free, open source, no API key. Results include pronunciation audio, definitions, synonyms, and source links.

```
Search any word →
  In curated list?  → Show DevStart entry + code example + SVG visual
  Not in list?      → Query Free Dictionary API → Show live results with audio
```

### Game Modes

<div align="center">

| Mode | Icon | What you do | Skill trained |
|------|:----:|-------------|--------------|
| **Code Fill-In** | 🧩 | Drag or click code pieces into blank spots | Recall & syntax |
| **Visual Decode** | 🎨 | Identify what each colour-highlighted part represents | Code reading |
| **Bug Hunt** | 🐛 | Click the one broken line before time runs out | Debugging instinct |

</div>

### Difficulty Stages

<div align="center">

| Stage | Timer | Focus |
|:-----:|:-----:|-------|
| 🟢 **Beginner** | `45s` | Doctype, mkdir, h1, git add — foundational syntax |
| 🔵 **Pro** | `30s` | Flexbox, arrow functions, -b flag, CSS selectors |
| 🔴 **Advanced** | `20s` | async/await, --amend, :first-child, nullish `??` |

</div>

### Sample Challenge — Bug Hunt (Pro)

```diff
  .container {
    display: flex;
    flex-direction: row;
-   justify-contents: center;   ← 🐛 bug on this line
+   justify-content: center;    ← ✓ fix: "contents" → "content"
    gap: 16px;
  }
```


Every round ends with an animated SVG score ring, accuracy %, best streak, a personalised tip, and automatic next-level suggestions when you hit ≥ 75%. Score ≥ 80% → confetti.

## Git Workflow Used

<div align="center">
<img src="https://konstantinfo.com/blog/wp-content/uploads/2023/12/Github-Branching-and-Version-Control-Explained.jpg" width="80%" alt="Git branching diagram" />
<br><em>The feature-branch workflow used throughout this project</em>
</div>

<br>

```bash
# ── Initialisation ──────────────────────────────────────────────
git init
git remote add origin https://github.com/edwinnyandika/gomycode-cli-assignment.git

# ── First commit ────────────────────────────────────────────────
git add .
git commit -m "feat: initial CLI assignment documentation"
git push -u origin main

# ── Feature branches ────────────────────────────────────────────
git checkout -b feature/interactive-quiz
git add .
git commit -m "feat: add 20-question quiz with sound and confetti"
git push origin feature/interactive-quiz

git checkout main
git merge feature/interactive-quiz
git push origin main

# ── Inspection commands used during development ─────────────────
git log --oneline --graph --all --decorate
git status
git diff
git stash && git stash pop
```

## Contributing

Personal project, but PRs, issues, and suggestions are genuinely welcome.

```bash
# 1. Fork on GitHub, then clone your fork
git clone https://github.com/YOUR_USERNAME/gomycode-cli-assignment.git
cd gomycode-cli-assignment

# 2. Create a descriptively named branch
git checkout -b fix/quiz-question-3-typo
git checkout -b feat/add-python-editor-snippet

# 3. Make changes to index.html
#    Test by opening in a browser or Live Server

# 4. Commit with a clear conventional prefix
git add .
git commit -m "fix: correct typo in bug-hunt level 3 pro"
git commit -m "feat: add Python hello-world to editor snippets"

# 5. Push and open a pull request
git push origin fix/quiz-question-3-typo
```

**Commit conventions:**

| Prefix | When |
|--------|------|
| `feat:` | New feature or section |
| `fix:` | Bug fix |
| `style:` | CSS / visual — no logic change |
| `refactor:` | Restructure — same behaviour |
| `docs:` | README or comments |
| `chore:` | Rename, housekeeping |
