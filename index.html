Step-by-Step Tasks
Each task below shows the exact command to run, a simulated terminal output, and a breakdown of what every part of the command means.
01
Create the GOMYCODE folder in home
cd ~  →  mkdir GOMYCODE

First, we navigate to the home directory — this is your personal root folder in the system (like C:/Users/YourName on Windows). Then we create a new folder called GOMYCODE using mkdir (make directory).

Git Bash — ~/
~ $ cd ~
~ $ mkdir GOMYCODE
~ $ ls
Desktop/   Documents/   Downloads/   GOMYCODE/
~ $
// command breakdown
cd Change Directory — moves you into a different folder. It's how you navigate the filesystem.
~ The tilde symbol is a shortcut for your home directory. cd ~ always takes you back home no matter where you are.
mkdir Make Directory — creates a new folder. The name you give it comes right after.
ls List — shows all files and folders in your current location. Use it to verify your command worked.
02
Navigate into GOMYCODE and create first-repo
cd GOMYCODE  →  mkdir first-repo

Now we enter the GOMYCODE folder we just created, and inside it, create a second folder called first-repo. Notice the hyphen — folder names with spaces can cause problems in CLI, so hyphens are commonly used instead.

Git Bash — ~/GOMYCODE
~ $ cd GOMYCODE
~/GOMYCODE $ mkdir first-repo
~/GOMYCODE $ ls
first-repo/
~/GOMYCODE $
// command breakdown
cd GOMYCODE Enters the GOMYCODE folder. Notice the prompt changes to ~/GOMYCODE — this confirms you're now inside it.
mkdir first-repo Creates the first-repo folder inside GOMYCODE. The hyphen is part of the name — it's not a flag.
03
Navigate into first-repo and create a .txt file with your last name
cd first-repo  →  touch lastname.txt

Enter first-repo, then use the touch command to create an empty text file named after your last name. In the example below, we use wick.txt — replace it with your actual last name.

Git Bash — ~/GOMYCODE/first-repo
~/GOMYCODE $ cd first-repo
~/GOMYCODE/first-repo $ touch wick.txt
~/GOMYCODE/first-repo $ ls
wick.txt
~/GOMYCODE/first-repo $
// command breakdown
touch Touch — creates a new, empty file. If the file already exists, it just updates the timestamp. It does NOT open or edit the file.
wick.txt The filename. .txt is the file extension, meaning it's a plain text file. Replace wick with your own last name.
04
Delete the file you just created
rm wick.txt

The rm command permanently deletes a file. Unlike dragging to the Recycle Bin, there is no undo. The file is gone immediately. Always double-check before running rm.

Git Bash — ~/GOMYCODE/first-repo
~/GOMYCODE/first-repo $ rm wick.txt
~/GOMYCODE/first-repo $ ls
(empty — no files)
~/GOMYCODE/first-repo $
// command breakdown
rm Remove — permanently deletes a file. No confirmation, no recycle bin. It's instant and irreversible.
wick.txt The exact name of the file to delete. Spelling must be precise — the terminal is case-sensitive.
05
Create a new .txt file with your first name
touch john.txt

Same command as Task 3 — but this time using your first name. Replace john with your actual first name.

Git Bash — ~/GOMYCODE/first-repo
~/GOMYCODE/first-repo $ touch john.txt
~/GOMYCODE/first-repo $ ls
john.txt
~/GOMYCODE/first-repo $
// command breakdown
touch john.txt Creates an empty file called john.txt. The .txt extension tells the OS it's a plain text file.
06
Create another file called index.md
touch index.md

Create a second file — this time with a .md extension. Markdown is a lightweight formatting language used by developers to write documentation. You'll see it everywhere in coding projects (like README files on GitHub).

Git Bash — ~/GOMYCODE/first-repo
~/GOMYCODE/first-repo $ touch index.md
~/GOMYCODE/first-repo $ ls
index.md   john.txt
~/GOMYCODE/first-repo $
// command breakdown
.md Markdown extension. Markdown files use simple syntax like # Heading and **bold** to format text. Used widely on GitHub.
ls Verify both index.md and john.txt now exist in the folder.
07
Write "Hello World" into index.md
echo "Hello World" > index.md

The echo command prints text. The > symbol redirects that output into a file instead of the screen. Together, echo "text" > file writes text directly into a file. We then use cat to verify the content was saved.

Git Bash — ~/GOMYCODE/first-repo
~/GOMYCODE/first-repo $ echo "Hello World" > index.md
~/GOMYCODE/first-repo $ cat index.md
Hello World
~/GOMYCODE/first-repo $
// command breakdown
echo Prints whatever text follows it to the terminal output.
"Hello World" The text to write. Quotes keep the two words together as one argument.
> Redirect operator — instead of printing to screen, it writes the output into a file. Warning: it overwrites the file if it already has content.
cat Concatenate — reads and displays the contents of a file in the terminal. Used here to verify the write worked.
08
Rename index.md to index.html
mv index.md index.html

The mv command means move — but when the destination is in the same folder, it effectively renames the file. No copy is made; the original index.md disappears and becomes index.html with all its content intact.

Git Bash — ~/GOMYCODE/first-repo
~/GOMYCODE/first-repo $ mv index.md index.html
~/GOMYCODE/first-repo $ ls
index.html   john.txt
~/GOMYCODE/first-repo $ cat index.html
Hello World
~/GOMYCODE/first-repo $
// command breakdown
mv Move — moves a file to a new location. When source and destination are in the same folder, it renames the file.
index.md The source — the file you want to rename/move.
index.html The destination — the new name. The content inside the file is preserved; only the name changes.
09
Copy index.html into a new README.md
cp index.html README.md

The cp command copies a file. Unlike mv, the original file stays. You end up with two files — index.html still exists, and a new README.md is created with identical content.

Git Bash — ~/GOMYCODE/first-repo
~/GOMYCODE/first-repo $ cp index.html README.md
~/GOMYCODE/first-repo $ ls
index.html   john.txt   README.md
~/GOMYCODE/first-repo $ cat README.md
Hello World
~/GOMYCODE/first-repo $
// command breakdown
cp Copy — duplicates a file. The original stays where it is, and a new copy is created at the destination.
index.html The source file to copy from. This file still exists after the command runs.
README.md The destination file name. A new file is created with the same content as index.html.
