Step-by-Step CLI Tasks
Each task below shows the exact command, a simulated terminal output, and a full breakdown. Try the Live Terminal below to practice yourself.
01
Create the GOMYCODE folder in home directory
cd ~ → mkdir GOMYCODE

Start by navigating to your home directory and creating a new folder called GOMYCODE. This will be the root of all your Go My Code work.

Git Bash — ~
~ $cd ~
~ $mkdir GOMYCODE
~ $ls
Desktop/   Documents/   Downloads/   GOMYCODE/
~ $
// command breakdown
cd ~Change Directory to Home — the tilde ~ is a shortcut for your home directory. No matter where you are, cd ~ brings you home.
mkdirMake Directory — creates a new folder. The name comes right after.
lsList — shows all files and folders in your current location. Always verify your work!
02
Navigate into GOMYCODE and create first-repo
cd GOMYCODE → mkdir first-repo

Enter the folder you just made, then create a sub-folder called first-repo. Hyphens in folder names are standard — spaces cause issues in the CLI.

Git Bash — ~/GOMYCODE
~ $cd GOMYCODE
~/GOMYCODE $mkdir first-repo
~/GOMYCODE $ls
first-repo/
~/GOMYCODE $
// command breakdown
cd GOMYCODEEnters GOMYCODE. Notice the prompt changes from ~ to ~/GOMYCODE — that's how you know you're inside.
mkdir first-repoCreates the first-repo folder inside GOMYCODE. The hyphen is part of the name — not a flag.
03
Navigate into first-repo and create a .txt file with your last name
cd first-repo → touch nyandika.txt

Use touch to create an empty text file named after your last name. It creates the file instantly without opening it.

Git Bash — ~/GOMYCODE/first-repo
~/GOMYCODE $cd first-repo
~/GOMYCODE/first-repo $touch nyandika.txt
~/GOMYCODE/first-repo $ls
nyandika.txt
~/GOMYCODE/first-repo $
// command breakdown
touchTouch — creates a new, empty file. If the file already exists, it just updates its timestamp. It does NOT open the file.
.txtThe file extension. It tells the OS this is a plain text file. Other extensions: .html, .css, .js
04
Delete the file you just created
rm nyandika.txt

The rm command permanently deletes a file — no Recycle Bin, no undo. Be careful.

Git Bash — ~/GOMYCODE/first-repo
~/GOMYCODE/first-repo $rm nyandika.txt
~/GOMYCODE/first-repo $ls
(empty — no files)
~/GOMYCODE/first-repo $
// command breakdown
rmRemove — permanently deletes a file. No confirmation, no recycle bin. Instant and irreversible.
rm -rf folder/To delete a folder and all its contents. The -r flag = recursive, -f = force. Use with extreme caution.
05
Create a new .txt file with your first name
touch edwin.txt

Now create a file with your first name instead. This keeps the repository and replaces the deleted file.

Git Bash — ~/GOMYCODE/first-repo
~/GOMYCODE/first-repo $touch edwin.txt
~/GOMYCODE/first-repo $ls
edwin.txt
~/GOMYCODE/first-repo $
// command breakdown
touch edwin.txtCreates edwin.txt in the current folder. Replace edwin with your own first name when doing this task.
06
Create an index.md file
touch index.md

Create a Markdown file called index.md. Markdown is a simple markup language used in README files, documentation, and GitHub pages.

Git Bash — ~/GOMYCODE/first-repo
~/GOMYCODE/first-repo $touch index.md
~/GOMYCODE/first-repo $ls
edwin.txt   index.md
~/GOMYCODE/first-repo $
// command breakdown
.mdMarkdown extension — Markdown files use special syntax like # Heading, **bold**, and [link](url) to format text.
07
Write "Hello World" into index.md using echo
echo "Hello World" > index.md

The echo command prints text. The > redirect operator sends that text into a file instead of the screen. This is how you write content from the CLI.

Git Bash — ~/GOMYCODE/first-repo
~/GOMYCODE/first-repo $echo "Hello World" > index.md
~/GOMYCODE/first-repo $cat index.md
Hello World
~/GOMYCODE/first-repo $
// command breakdown
echoPrints text to the terminal output.
>Redirect operator — sends output INTO a file. Overwrites existing content. Use >> to append instead.
catConcatenate/display — prints the contents of a file to the terminal. Great for quick checks.
08
Rename index.md to index.html
mv index.md index.html

mv means move but when used within the same folder, it renames the file. When you change the extension from .md to .html, the file type changes.

Git Bash — ~/GOMYCODE/first-repo
~/GOMYCODE/first-repo $mv index.md index.html
~/GOMYCODE/first-repo $ls
edwin.txt   index.html
~/GOMYCODE/first-repo $
// command breakdown
mvMove / Rename — moves a file to a new path, OR renames it. Syntax: mv [source] [destination]
mv file.txt ../Move to the parent directory. Combining rename + move is also possible.
09
Copy index.html to README.md
cp index.html README.md

cp copies a file. The original stays. The copy gets a new name or goes to a different location. GitHub automatically displays README.md on your repository page.

Git Bash — ~/GOMYCODE/first-repo
~/GOMYCODE/first-repo $cp index.html README.md
~/GOMYCODE/first-repo $ls
edwin.txt   index.html   README.md
~/GOMYCODE/first-repo $
// command breakdown
cpCopy — creates a duplicate of a file. Unlike mv, the source file remains. Syntax: cp [source] [destination]
cp -r folder/ backup/Copy an entire folder. The -r flag means recursive — includes all files inside.

Setup & Init
These are the first commands you run when starting with Git — configuring your identity and creating/cloning repositories.
git initsetup
Initializes a brand-new Git repository in the current folder. Creates a hidden .git/ folder that tracks all your changes.
$ git init
Initialized empty Git repository in ~/project/.git/
git clonesetup
Downloads a complete copy of a remote repository to your machine — including all history, branches, and files.
$ git clone https://github.com/user/repo.git
Cloning into 'repo'...
git configsetup
Configure your Git identity. This is stored in every commit you make — run this once when setting up Git on a new machine.
$ git config --global user.name "Edwin"
$ git config --global user.email "you@mail.com"
// core workflow
Staging & Committing
The three-step Git workflow: modify → stage → commit. These commands are used every single day.
git statusinfo
Shows the current state of your working directory — which files are modified, staged, or untracked. Always run this before committing.
$ git status
modified: index.html
untracked: style.css
git addstaging
Stages file changes — adds them to the "waiting area" ready to be committed. Use git add . to stage everything.
$ git add index.html
$ git add .  # stage all changes
git commitsaving
Saves a snapshot of your staged changes with a descriptive message. Commits are the permanent record of your project's history.
$ git commit -m "Add homepage layout"
[main a1b2c3] Add homepage layout
git loghistory
Shows the full commit history. Each entry includes a commit hash, author, date, and message. Use --oneline for a compact view.
$ git log --oneline
a1b2c3d Add homepage layout
e4f5g6h Initial commit
git diffinfo
Shows the exact line-by-line changes in modified files that haven't been staged yet. Green = added, red = removed.
$ git diff
+ <h1>Hello World</h1>
- <h1>Old Title</h1>
git restoreundo
Discards changes in a file and restores it to the last committed version. This is the modern replacement for git checkout -- file.
$ git restore index.html
# file reverted to last commit
// parallel development
Branching & Merging
Branches let you work on features without breaking the main code. Merge when you're ready.
git branchbranches
Lists all local branches. The active branch has a * next to it. Use git branch [name] to create a new branch.
$ git branch
* main
  feature/navbar
git switchbranches
Switch to a different branch. Use -c flag to create and switch to a new branch in one command.
$ git switch feature/navbar
$ git switch -c new-feature
git mergemerging
Combines changes from another branch into your current branch. First switch to main, then merge the feature branch.
$ git switch main
$ git merge feature/navbar
Merge made by 'ort' strategy.
// github & remote
Remote Repositories
Connect your local repo to GitHub. Push your code, pull updates, collaborate with others.
git remote addremote
Links your local repo to a remote on GitHub. The name origin is the standard alias for your remote.
$ git remote add origin https://github.com/user/repo.git
git pushremote
Uploads your local commits to GitHub. First push requires -u origin main to set the upstream branch.
$ git push -u origin main
$ git push  # subsequent pushes
git pullremote
Downloads and merges the latest changes from the remote branch into your current local branch. Runs fetch + merge together.
$ git pull
Already up to date.
git fetchremote
Downloads remote changes but does NOT apply them. Lets you review changes before merging. Safer than git pull.
$ git fetch origin
$ git diff main origin/main
// end-to-end example

This is what a full Git workflow looks like — from creating a project to pushing it to GitHub.

Anatomy of an HTML Element
Every HTML element follows the same pattern: an opening tag, content, and a closing tag. Here's how it breaks down.
</>
The anatomy of an HTML element
opening tag · content · closing tag · attributes
HTML Structure
// A complete HTML element:
<a href="https://github.com">Visit GitHub</a>
<a                    ↑ Opening Tag href="url"  ↑ Attribute Visit GitHub ↑ Content </a> ↑ Closing Tag
// element rules
Opening TagStarts with <, has the element name, and ends with >. Example: <p>
Closing TagSame as opening but with a forward slash. Example: </p>. Self-closing tags like <img> don't need one.
AttributeExtra info about an element, always inside the opening tag. Format: name="value". Multiple attributes are separated by spaces.
ContentWhat appears between the opening and closing tags. Can be text, images, or other HTML elements (nesting).

HTML Key Terms
Every term a beginner needs to know — with clear definitions and live examples. Click a card to explore.
<>
HTML Tag
Structure
A tag is the basic unit of HTML, wrapped in angle brackets. Tags tell the browser what type of content to display. Most tags come in pairs: opening and closing.
<h1>Main Heading</h1>
<p>Paragraph text</p>
@
Attribute
Modifier
Attributes provide extra information about an element. They always live inside the opening tag as key-value pairs. Some are required, others optional.
<img src="photo.jpg" alt="A photo">
[]
Element
Complete Unit
An element is the complete package: opening tag + content + closing tag. Think of it as a container. Tags define element type; elements hold the actual content.
<button class="btn">Click Me</button>
#
ID
Unique Identifier
The id attribute gives an element a unique name on the page. IDs must be unique — only one element per page can have the same ID. Used for CSS targeting and anchor links.
<section id="about">...</section>
.
Class
Group Label
The class attribute labels elements for CSS styling. Unlike IDs, many elements can share the same class — it's for grouping related elements.
<p class="highlight">Text</p>
<p class="highlight">More</p>
🔗
Hyperlink
Navigation
Links are created with the <a> (anchor) tag. The href attribute sets the destination. Use target="_blank" to open in a new tab.
<a href="https://github.com"
   target="_blank">GitHub</a>
▣
Block vs Inline
Display Types
Block elements (like <div>, <p>) take up the full width and start on a new line. Inline elements (like <span>, <a>) flow within text.
<div>Block</div>
<span>Inline</span>
🌲
DOM
Document Object Model
The DOM is a tree-like structure the browser builds from your HTML. Every element becomes a "node" in this tree. JavaScript can access and modify the DOM to make pages interactive.
html
 ├── head
 │   └── title
 └── body
     └── h1, p, div...
📄
Semantic HTML
Best Practice
Semantic tags describe what their content means, not just how it looks. They help search engines and screen readers understand your page structure.
<header> site header </header>
<main> main content </main>
<footer> footer </footer>
💬
Comment
Developer Note
Comments are notes in your code that the browser ignores. They're for developers to explain what's happening. Always comment complex sections of your HTML.
<!-- This is a comment -->
<!-- Navigation section -->
<nav>...</nav>

Standard HTML Page Structure
Every HTML file starts with this boilerplate. Memorize it — you'll type it thousands of times.
index.html — boilerplate
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Page</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <!-- Your content goes here -->
  <h1>Hello World</h1>
  <script src="script.js"></script>
</body>
</html>
